# 产品概述

## 项目定位

`caddy-anytls` 是一个运行在 Caddy 内部的 AnyTLS 接入模块。它的设计目标是在不改变现有站点部署方式的前提下，将 AnyTLS 接入同一个 `443` 入口，并继续复用 Caddy 的 TLS 握手、证书管理和站点路由能力。

从部署视角看，它不是独立代理服务，而是 Caddy 的一部分。运维侧仍然围绕 Caddy 配置和发布，AnyTLS 作为同一入口下的附加能力启用。

## 设计目标

当前版本聚焦以下目标：

1. 网站流量与 AnyTLS 流量共用同一个公网入口。
2. TLS 证书的申请、续期和装载继续由 Caddy 负责。
3. AnyTLS 服务端不再以独立守护进程形式部署。
4. 非 AnyTLS 流量保持原有网站访问路径，不影响现有 Web 业务。
5. 为 AnyTLS 出站目标提供保守默认策略和可配置的放行/拒绝边界。

## 非目标

当前版本不覆盖以下范围：

- 设计或实现新的代理协议
- 替代 Caddy 的 HTTP 站点能力
- 自行管理证书文件或 ACME 生命周期
- 引入多协议网关能力
- 解决 CDN 兼容性问题
- 要求额外部署 `caddy-l4`

## 适用场景

### 已有 Caddy 站点，希望接入 AnyTLS

如果现有服务已经由 Caddy 托管，并且 `443` 入口、证书和站点路由都已稳定运行，可以通过该模块在同一入口上增加 AnyTLS 能力，而不需要额外拆分 TLS 终止层。

### 希望统一网站与代理的入口管理

对于同时运营站点和代理入口的部署环境，该模块可以减少端口、证书和进程层面的重复配置，并将入口控制集中在 Caddy 侧。

### 希望代理异常时不影响站点访问

该模块会在 TLS 解密后识别协议；如果流量不是 AnyTLS，则连接继续回到网站链路。普通浏览器访问不会因为 AnyTLS 接入而改变处理路径。

## 核心能力

### 单端口共存

网站和 AnyTLS 共用 `443` 监听端口，不要求单独暴露额外 TLS 端口。

### 证书体系复用

证书继续由 Caddy 和 CertMagic 负责管理。模块本身不提供 `cert_file`、`key_file` 等独立证书配置。

### TLS 后识别

协议识别发生在 TLS 握手之后、HTTP 请求解析之前。模块处理的是已经解密的应用层字节流，而不是裸 TCP 或明文 HTTP 之前的流量。

### 网站回落

当连接不符合 AnyTLS 特征时，流量会继续交给 Caddy 原有的网站处理链路。

### 多用户、转发与出站策略

当前版本支持多用户配置、基础出站连接建立，以及 `UDP over TCP v2` 目标处理。默认拒绝访问私网、回环和链路本地目标，并可按端口、域名和 CIDR 组合限制出站目标。

### 配置重载

模块遵循 Caddy 的配置生命周期。配置更新后，新连接按照新配置处理；已有 AnyTLS 会话在旧配置卸载时会被终止，以保证策略边界清晰。

## 工作模型

连接处理过程可概括为：

1. 客户端连接到 Caddy 的 `443` 入口。
2. Caddy 根据站点配置完成 TLS 握手和证书选择。
3. 模块在 TLS 解密后读取首包并识别协议。
4. 若判定为 AnyTLS，则进入认证、目标连接建立和转发流程。
5. 若判定为非 AnyTLS，则连接继续进入网站链路。

这一模型的关键在于职责分离：

- Caddy 负责监听、TLS 和站点路由。
- `caddy-anytls` 负责协议识别、接入控制和会话桥接。
- AnyTLS 协议实现复用上游 `sing-anytls`。

## 运维价值

对部署侧而言，该模块带来的主要收益包括：

- 减少单独代理进程的维护成本
- 减少额外 TLS 入口和证书配置
- 将站点与代理的入口配置集中到同一套 Caddy 生命周期中
- 在配置重载和日志输出上维持一致的运维方式

## 配置原则

配置体验应聚焦于接入，而不是要求使用者理解完整网络栈。典型配置通常只需要：

- 站点域名
- 至少一个用户
- 必要的超时、并发或访问控制参数

模块不提供以下类型的配置项：

- 独立证书路径
- 独立 TLS 监听配置
- 自身的 ACME 配置

这些能力应继续由 Caddy 统一负责。

## 安全与行为边界

当前版本明确采用以下行为约束：

- 默认拒绝访问常见私网目标地址
- 域名目标会先解析再执行私网和 CIDR 策略，避免域名绕过私网保护
- `deny_*` 策略优先于 `allow_*` 策略
- 用户被禁用后，命中新连接会被拒绝，不回落到网站
- 配置卸载或重载时，旧 AnyTLS 会话会被终止
- 默认审计日志不输出密码；显式开启 `log_node_info` 时会输出包含密码的节点 URI，部署方需要控制日志访问权限

这些行为属于当前版本的实现边界，应在对外说明中保持一致。

## 当前范围

当前版本已经覆盖核心接入能力，但范围仍限定在接入层。以下能力不在本阶段范围内：

- 管理接口
- Prometheus 指标导出
- 用户级限速和限并发
- 跨配置代际的 AnyTLS 会话保活

## 验收标准

当前版本以以下结果作为成立条件：

1. 网站可以继续通过 `443` 提供服务。
2. AnyTLS 客户端可以通过同一入口完成连接和认证。
3. TLS 证书继续由 Caddy 自动管理。
4. 非 AnyTLS 请求能够稳定回落到网站。
5. 多用户配置能够生效。
6. 基础结构化日志可用于审计与排障。
7. 配置重载后，新连接能够按新配置处理。
8. 出站目标策略能够拒绝高风险目标，并允许按需精确放行。
